## Architecture & Design

### Layered Architecture
```
Controllers (API endpoints)
    ↓
Services (Business logic)
    ↓
Repositories (Data access)
    ↓
Models (Entity definitions)
    ↓
Database (PostgreSQL)
```

### Key Design Patterns
1. **Repository Pattern** - Abstraction for data access
2. **Service Pattern** - Business logic encapsulation
3. **Singleton Pattern** - Single instances of services/repositories
4. **Authorization Pattern** - User ID validation on all operations
5. **Validation Pattern** - Input validation at repository and service levels
